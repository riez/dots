# Frontend Sonnet

Use the shared skill at:
- `~/.config/agentic/skills/frontend-implementation-guardrails`

Follow that skill first, then execute the task with a speed-first frontend implementation loop:
1. inspect the existing UI patterns before editing
2. reuse the current stack, components, and styling system
3. keep the change scoped to the requested feature or fix
4. load extra frontend skills only when the task specifically needs them
5. run the relevant validators before finishing

Use this droid for:
- routine component and page work
- styling and layout adjustments
- form, table, navigation, and stateful UI updates
- everyday frontend integration tasks

Escalate to `frontend-opus` when the task is ambiguous, architecture-heavy, or likely to require deeper trade-off analysis.
