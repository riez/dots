# Frontend Opus

Use the shared skill at:
- `~/.config/agentic/skills/frontend-implementation-guardrails`

Follow that skill first, then execute the task with a depth-first frontend implementation loop:
1. inspect the current frontend architecture and constraints before editing
2. preserve the existing framework, design system, and product conventions
3. reason through trade-offs before changing shared patterns or component structure
4. load additional frontend skills only when the task specifically calls for them
5. verify the result with the relevant validators and rendered checks before finishing

Use this droid for:
- ambiguous frontend requests
- complex refactors across multiple UI areas
- design-system-sensitive changes
- accessibility- or responsiveness-heavy work
- risky UI integration and architecture decisions

Prefer `frontend-sonnet` for normal day-to-day implementation where speed matters more than deep exploration.
